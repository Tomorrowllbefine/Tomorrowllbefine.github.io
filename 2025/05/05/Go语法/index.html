switch

Go 的 switch 语句类似于 C、C++、Java、JavaScript 和 PHP 中的，不过 Go 只会运行选定的 case，而非之后所有的 case。

几点区别：

  1. 无需显示break：在效果上，Go 的做法相当于这些语言中为每个 case 后面自动添加了所需的 break 语句。

  2. fallthrough可实现执行后续case：在 Go 中，除非以 fallthrough 语句结束，否则分支会自动终止。

  3. switch 的 case 无需为常量，且取值不限于整数；

Q：如何选择函数的接收者？

选择指针接收者的原因：

  • 方法可以修改其接收者指向的值；
  • 避免每次方法调用时复制该值；

通常来说，所有给定类型的方法都应该有值或指针接收者，但并不应该二者混用。

接口值

在Go内部，接口可以看作是包含“值”和“具体类型”的元组（value, type）：

  • value：接口内部保存了实现该接口的类型的实例；这个值可能是某个结构体实例，也可能是其它类型的值；

> 接口值保存了一个具体底层类型的具体值。接口值调用方法时会执行其底层类型的同名方法；

> 接口值可以用作函数的参数或返回值；

  • type：接口内部保存了该接口值的实际类型信息。接口值包含了内部存储的数据的“具体类型”是什么，即实现了该接口所有方法的具体类型；

类型断言

类型断言：提供了访问接口值底层具体值（实现）的方式；

  • 形式：t := i.(T)；

  • 含义：断言 “接口值i保存了具体类型T，并将其底层类型为T的值赋予变量t” ；

若i并未保存T类型的值，该语句就会触发一个panic；

为了 判断 一个接口值是否保存了一个特定的类型，类型断言可返回两个值：其底层值以及一个报告断言是否成功的布尔值。

分析：

  • 若 i 保存了一个 T，那么 t 将会是其底层值，而 ok 为 true；

  • 否则，ok 将为 false 而 t 将为 T 类型的零值，程序并不会产生 panic；

类型选择

类型选择：是一种按顺序从几个类型断言中选择分支的结构；

类型选择与一般的 switch 语句相似，不过类型选择中的 case 为类型（而非值），它们针对给定接口值所存储的值的类型进行比较；

类型选择中的声明与类型断言 i.(T) 的语法相同，只是具体类型 T 被替换成了关键字 type。

此选择语句判断接口值 i 保存的值类型是 T 还是 S。在 T 或 S 的情况下，变量 v 会分别按 T 或 S 类型保存 i 拥有的值。在默认（即没有匹配）的情况下，变量 v 与 i 的接口类型和值相同。
